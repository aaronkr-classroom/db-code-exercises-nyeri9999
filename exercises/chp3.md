## 연습 문제

### 01. 요구 수집 단계의 주요 업무가 아닌 것은?
1. 수용 범위 결정
2. 기존 문서 조사
3. 사용자 요구사항 수집
4. 법기관적 제약조건 파악

**정답:** 1번  
→ 수용 범위 결정은 **요구 분석 단계**에 해당한다.

---

### 02. 요구 수집 명세서 작성 시 포함해야 할 항목에 해당하지 않는 것은?
1. 수집한 세부 내용을 포함하는 첨부 자료
2. 업무 영역 분할도
3. 정보 제공자
4. 수집 담당자

**정답:** 2번  
→ 업무 영역 분할도는 **요구 분석 단계 산출물**이다.

---

### 03. 요구 분석 단계의 주요 업무가 아닌 것은?
1. 업무 영역 분할
2. 상세 업무 분석
3. 트랜잭션 요구 분석 명세서 작성
4. 사용자 그룹 및 담당자 식별

**정답:** 4번  
→ 사용자 그룹 및 담당자 식별은 **요구 수집 단계**에 해당한다.

---

### 04. 요구 분석이 어려운 이유
**정답:**
1. 개발자와 사용자 간 의사소통이 어렵다.
2. 요구사항이 계속 변화한다.
3. 요구사항 문서화가 어렵다.

---

### 05. 요구 수집을 위한 인터뷰 시 유의 사항으로 옳지 않은 것은?
1. 기분 상하지 않게 주의
2. 정해진 시간 내 종료
3. 닫힌 질문과 열린 질문 혼합
4. 인터뷰 내용을 미리 알려 선입견을 갖게 함

**정답:** 4번  
→ 인터뷰 대상자에게 **선입견을 갖게 하면 안 된다.**

---

### 06. DB 구축 전체 단계에서 요구 분석이 가장 중요한 이유
**정답:**  
요구 분석 단계에서 오류가 발생하면 이후 설계와 구현 단계에 많은 시간과 비용을 투입하더라도 원하는 시스템을 구축하기 어렵기 때문이다.

---

### 07. 요구사항의 특징에 해당하지 않는 것은?
1. 계속 변화
2. 완전하다
3. 라이프사이클 존재
4. 점차 확장

**정답:** 2번  
→ 요구사항은 일반적으로 **불완전할 수밖에 없다.**

---

### 08. 요구사항 분석 시 고려 사항으로 옳지 않은 것은?
1. 현재 필요한 요구사항 중심
2. 상반 요구사항 조정

**정답:** 1번  
→ 현재뿐 아니라 **가까운 미래의 요구사항도 예측**해야 한다.

---

### 09. 데이터 요구 분석 명세서 작성 시 유의 사항으로 옳은 것은?
1. 전문 용어 사용

# 02. 트랜잭션 요구 분석 명세서

트랜잭션은 **검색(Search) / 삽입(Insert) / 삭제(Delete) / 변경(Update)** 형태로 작성한다.

---

## 1) 여행상품 관리

| 주체 | 작업 | 트랜잭션 유형 |
|---|---|---|
| 여행상품 담당자 | 신규 여행상품 정보를 삽입한다 | Insert |
| 여행상품 담당자 | 여행상품 정보를 변경한다 | Update |
| 여행상품 담당자 | 여행상품 정보를 검색한다 | Search |
| 여행상품 담당자 | 여행상품 정보를 삭제한다 | Delete |

---

## 2) 예약 관리

| 주체 | 작업 | 트랜잭션 유형 |
|---|---|---|
| 고객 | 여행상품 예약 정보를 삽입한다 | Insert |
| 고객 | 예약 정보를 취소한다 | Delete |
| 시스템 | 24시간 미결제 예약을 삭제한다 | Delete |
| 시스템 | 출발 취소 시 환불 정보를 변경한다 | Update |

---

## 3) 고객 관리

| 주체 | 작업 | 트랜잭션 유형 |
|---|---|---|
| 고객관리 직원 | 고객 정보를 삽입한다 | Insert |
| 고객관리 직원 | 고객 등급을 변경한다 | Update |
| 고객관리 직원 | 고객 예약 내역을 검색한다 | Search |

---

## 4) 직원 관리

| 주체 | 작업 | 트랜잭션 유형 |
|---|---|---|
| 관리자 | 직원 정보를 삽입한다 | Insert |
| 관리자 | 담당 업무를 변경한다 | Update |
| 관리자 | 직원 정보를 검색한다 | Search |
