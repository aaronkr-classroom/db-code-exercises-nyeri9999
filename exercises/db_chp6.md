## 연습문제

1. 디음 중 개체 심화 요소에 대한 설명으로 적절하지 않는 것은?
  1. 키 개체는 다른 개체의 부모 개체가 되므로 가장 먼저 추출해야 한다.
  2. 관계 개체는 ER 모텔에서 대개 관계에 해당한다.
  3. 코드 개체는 정보를 간단히 표현하기 위한 기호를 나타내는 개체다.
  4. 약한 개체는 소유 개체가 있어야 존재할수 있는 개체다.

답: 2번

2. 키 개체와 메인 개체의 차이점을 설명하시오.

답: 키 개체는 업무에서 원래부터 독립적으로 존재하는 개체이고, 다른 개체의 부모 개체가 된다. 반면 메인 개체는 키 개체 간의 업무적 관련성 때문에 생성되는 개체로, 부모 개체를 갖는 파생 개체이지만 업무의 중심 역할을 한다.

3. 관계 개체와 메인 개체의 차이점을 설명하시오

답: 관계 개체는 두 개체 간의 M:N 관계를 표현하기 위해 생성되는 개체이다. 반면 메인 개체는 키 개체 간의 업무적 관련성 때문에 생성되는 개체로, M:N뿐만 아니라 1:1, 1:N 관계에서 생성되는 핵심 업무 개체까지 포함한다.

4. 다음과 같은 병렬 관계를 직렬 관계로 변환하여 ER 다이어그램으로 표현하시오. (슬라이드 나 책 235쪽)

답:

```text
[부서] M ─── 관리하다 ─── N [공사]
              |
           관리유형
        (발주, 감독, 정산)
```

답: 설계

6. 슈퍼 서브 타업을 정제할 때 어떤 경우에 슈퍼타입 개체를 기준으로 통합하는지 설명하시오.

답: 응용 프로그램에서 서브타입을 구분해서 처리하지 않는 경우, 서브타입 데이터를 명확히 구분하기 어려운 경우, 서브타입 개체에 포함되는 속성 수가 매우 적은 경우에는 슈퍼타입 개체를 기준으로 통합한다.

7. 일반회원과 VIP회원을 처리하는 모든응용 프로그램이 병확히 구분되고， 추후 추가될 응용프로그램도 마찬가지다. 아래 슈퍼-서브 타입을 바르게 정제한 결과를 ER 다이어그램으로 표현하시오.

답:

```text
[일반회원]
- 회원번호(PK)
- 이름
- 주소
- 연락처
- 일반회원 관련 속성

[VIP회원]
- 회원번호(PK)
- 이름
- 주소
- 연락처
- VIP회원 관련 속성
```

8. 사원 가운데 일부는 관리자이므로 유일한 ‘관리자변호’와 ‘직책’， ‘임명일자’， ‘임명기간’ 속성이 필요하여 아래와 같이 사원 개체를 표현했는데， 사원 가운데 관리자의 비율은 5% 미만이다. 이 ER 델을 좀 더 효율적으로 정제해서 ER 다이어그램으로 표현하시오.

답:

```text
[사원] 1 ─── 관리자이다 ─── 0..1 [관리자]

[사원]
- 사원번호(PK)
- 이름
- 주소
- 연락처

[관리자]
- 사원번호(PK, FK)
- 관리자번호
- 직책
- 임명일자
- 임명기간
```

10. 다음 중 속성을 검증하는 방법으로 적절하지 않는 것은?
  1. 각각 의미가 다른 여러 개의 속성을 묶어서 하나의 속성으로 정의한 것이라면 다중치 속성으로 표현해야 한다.
  2. 모든 개체가 통일한 하나의 값만 갖는 속성이 있다면 제거해야 한다.
  3. 코드 값을 갖는 속성이 현업에서 통상적으로 시용되는 코드 값이 아니면 코드 사용을 삼가야 한다.
  4. 전산 처리에 필요한 플래그flag 형태의 속성은 제외해야 한다.

답: 4번
