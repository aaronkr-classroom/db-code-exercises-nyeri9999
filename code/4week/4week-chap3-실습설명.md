# 간단 설명 파일입니다. 2544402 고다니엘
## STEP 1. 시스템 선택(저번 주와 다름)
* 선택한 시스템: 게임 유저 관리 시스템
* 시스템 간단 설명: 게임에 가입한 유저들의 정보를 저장하고 관리하는 시스템입니다.
## STEP 2. 요구사항 작성(조건: 사용자 2명 이상, 요구사항 5개 이상)
### 사용자 정의
* **관리자(Admin)**
   * 전체 유저 조회
   * 유저 정보 수정
   * 유저 삭제
   * 랭킹 조회
* **일반 유저(Player)**
   * 회원가입
   * 자신의 정보 조회
   * 플레이 기록 확인 
### 요구사항 정의
   1. 유저는 회원가입할 수 있다.
   2. 유저의 레벨 정보를 저장한다.
   3. 유저의 게임머니를 저장한다.
   4. 유저의 플레이 시간을 저장한다.
   5. 유저 랭크를 저장한다.
   6. 관리자는 전체 유저 목록을 조회할 수 있다.
   7. 특정 레벨 이상 유저를 검색할 수 있다.
   8. 게임머니 순으로 정렬하여 조회할 수 있다.
## STEP 3. 데이터 설계 (조건: 엔티티 2개 이상)
* 순서대로 엔티티 속성명, 속성 타입, 속성 역할입니다.
* 관계 설명:
  * 한 명의 유저는 하나의 게임 프로필을 가짐 -> 1:1
  * User.user_id ↔ GameProfile.user_id
* **User 엔티티: 유저 기본 정보 저장**
    * user_id / SERIAL / 유저 번호 (PK)
    * username / VARCHAR(30) / 유저 이름
    * email / VARCHAR(50) / 이메일
    * join_date / DATE / 가입일
* **GameProfile 엔티티: 게임 플레이 정보 저장**
    * profile_id / SERIAL	/ 프로필 번호 (PK)
    * user_id / INT / 유저 번호 (FK)
    * level / INT / 레벨
    * gold / INT / 게임머니
    * play_time / INT / 플레이 시간
    * rank_tier / VARCHAR(20) / 랭크
